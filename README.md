# 本地全能转文字

一个以安卓 App 为主的本地转文字工具，支持在安卓手机上进行视频转文字、语音转文字和图片转文字。识别流程尽量在设备本地完成，减少音视频、图片和文字内容上传到云端的隐私风险。

## 功能

- 视频转文字：从本地视频中提取音频并转写为文本
- 语音转文字：使用本地语音识别模型完成离线转写
- 图片转文字：使用 OCR 识别图片中的文字
- 本地处理：核心识别资源随 App 或本地模型文件提供
- 隐私保护：用户文件默认不上传到远程服务器

## 技术栈

- Kotlin
- Android Jetpack Compose
- Vosk Android
- sherpa-onnx
- Google ML Kit Text Recognition

## 本地构建

1. 使用 Android Studio 打开项目根目录。
2. 确认已安装 Android SDK 35 和 JDK 17。
3. 同步 Gradle。
4. 运行 `app` 模块，或执行：

```powershell
.\gradlew :app:assembleStandardDebug
```

## 全离线翻译版

项目已加入 `fullOffline` 构建版本，用于把本地翻译模型内置进 APK，并通过 ONNX Runtime Android 执行 Marian 推理。

当前已内置并接入：

- `en-zh`：英文转中文
- `zh-en`：中文转英文
- `en-ja`：英文转日文
- `ja-en`：日文转英文
- `ko-en`：韩文转英文

当前通过中转支持：

- 中文 ↔ 日文：通过英文中转
- 韩文 -> 中文：通过英文中转

暂不开放韩文作为目标语言，因为当前未内置英文转韩文 ONNX 模型。APP 会在翻译弹窗里禁用不可用目标，避免误触失败路线。

注意：接入 ONNX Runtime 后，最低 Android 版本调整为 Android 7.0（API 24）。

模型目录规范见 [`docs/full-offline-translation.md`](docs/full-offline-translation.md)。

检查模型文件：

```powershell
.\gradlew :app:checkFullOfflineTranslationModels
```

打包全离线版：

```powershell
.\gradlew :app:assembleFullOfflineDebug
```

## 转换首页

首页提供视频、语音、图片三类入口。用户选择文件或开始录音后，应用会显示当前转换状态、进度和最新结果。

首页还包含本地模型状态和性能模式设置：

- 精准语音模型已内置时，会提示可以直接开始识别。
- 如果模型缺失，可以使用备用导入功能导入本地语音模型压缩包。
- 性能模式会影响语音分块长度、识别线程数、图片缩放尺寸和长图切片高度。

## 视频转文字

视频转文字用于把本地视频里的语音内容转为文字，并生成可校对的字幕片段。

主要流程：

1. 点击视频转文字。
2. 选择识别语言。
3. 选择手机本地视频文件。
4. 应用在后台提取视频音频。
5. 使用本地语音识别模型转写。
6. 自动保存结果到历史记录。

视频转换结果包含：

- 完整识别文本。
- 分段字幕数据。
- 原视频路径。
- 转换状态和创建时间。

如果转换完成后进入历史记录，可以点击视频记录里的“编辑字幕”进入字幕校对界面。

## 语音转文字

语音转文字支持两种来源：

- 实时录音：直接使用手机麦克风识别。
- 导入音频：选择本地音频文件后转写。

实时录音能力：

- 开始录音。
- 暂停录音。
- 继续录音。
- 保存识别内容到历史记录。
- 清空当前草稿。
- 连续录音开关：系统停止后可自动继续聆听，适合较长时间口述。

导入音频时，应用会把音频分块处理，降低一次性读取大文件导致的内存压力。

## 图片转文字

图片转文字用于识别相册、截图、文档照片和长图中的文字。

主要能力：

- 选择本地图片后自动识别文字。
- 自动保存 OCR 结果到历史记录。
- 根据性能模式缩放图片，减少内存占用。
- 长图会分块切片识别，降低内存溢出风险。

图片 OCR 使用 Google ML Kit Text Recognition。识别输入和输出以设备端处理为目标，但 ML Kit 可能根据其条款进行模型更新、兼容性信息或性能指标通信，正式发布时应在隐私说明中保留相关说明。

## 识别语言

语音和视频转文字支持以下识别语言：

- 普通话
- 英文
- 日语
- 韩语
- 粤语
- 其它方言

“其它方言”用于四川话、上海话、口音普通话、中英混合等更复杂场景。

## 性能模式

应用提供三种性能模式：

| 模式 | 适用场景 |
| --- | --- |
| 省内存 | 低内存手机优先，速度慢一点，但更稳定。 |
| 均衡 | 默认推荐，在稳定性和速度之间折中。 |
| 高性能 | 新手机优先，速度更快，但更占用内存。 |

性能模式会影响：

- 语音识别线程数。
- 音频分块秒数。
- 图片最大边长。
- 长图 OCR 切片高度。

## 后台转换

视频、音频和图片转换已接入后台服务。开始转换后，即使用户切到后台，转换也会继续进行。

后台转换特点：

- 使用前台服务通知显示转换状态。
- 通知中显示进度。
- 支持取消后台转换。
- 使用 Wake Lock 降低息屏后任务被系统暂停的概率。
- 转换完成后结果自动写入历史记录。

## 历史记录

历史记录会自动保存每次转换结果。

支持的操作：

- 查看全部历史。
- 按图片、语音、视频筛选。
- 复制识别文本。
- 分享识别文本。
- 删除单条记录。
- 清空全部历史记录。
- 视频记录可进入字幕校对。

历史记录中会显示转换类型、状态、创建时间和文本预览。

## 字幕校对与视频播放器

视频转换成功后，可以进入字幕校对界面。该界面把播放器和字幕列表放在一起，方便边看视频边修正文字。

播放器能力：

- 播放和暂停。
- 拖动进度条。
- 快退 10 秒。
- 快进 10 秒。
- 倍速播放，支持 0.5x、0.75x、1x、1.25x、1.5x、2x。
- 自动、竖屏、横屏三种方向模式。
- 全屏播放。
- 显示或隐藏字幕。
- 原文、译文、双语字幕显示模式。

字幕编辑能力：

- 点击单条字幕进行文字修改。
- 删除字幕片段。
- 批量提前 0.5 秒。
- 批量延后 0.5 秒。
- 保存修改后的字幕。
- 复制完整字幕文本。
- 分享完整字幕文本。

如果原视频文件无法读取，可以在字幕校对界面重新选择视频文件。

## 全屏历史视频切换

全屏播放器右上角提供历史视频按钮，可打开已转换视频列表。

该功能用于：

- 查看已经转换过的视频记录。
- 显示视频缩略图、标题和字幕段数。
- 快速切换到其它已转换视频。
- 当前字幕有未保存修改时，提示保存并切换、不保存切换或取消。

## 本地字幕翻译

fullOffline 版内置 OPUS-MT / Marian ONNX 量化翻译模型，并通过 ONNX Runtime Android 在本地执行字幕翻译。

翻译入口：

1. 打开视频字幕校对界面。
2. 点击“翻译字幕”。
3. 选择原语言和目标语言。
4. 选择显示方式。
5. 必要时点击“启动模型”预热当前翻译路线。
6. 点击“开始翻译”。

显示方式：

- 只看译文。
- 双语字幕。

当前已内置翻译方向：

- 英文 -> 中文
- 中文 -> 英文
- 英文 -> 日文
- 日文 -> 英文
- 韩文 -> 英文

当前通过中转支持：

- 中文 -> 日文，通过英文中转。
- 日文 -> 中文，通过英文中转。
- 韩文 -> 中文，通过英文中转。

当前限制：

- 暂未内置英文 -> 韩文模型，因此韩文暂不作为目标语言开放。
- 标准版不包含本地字幕翻译模型，需要使用 fullOffline 版。
- 翻译质量取决于内置模型能力，适合字幕辅助理解，正式发布字幕建议人工复核。
